2. HIGH-LEVEL MVP LOGIC
==================================================

For each item of the selected supplier:

A) Take current stock data from PS365 for warehouse store 777:
- stock
- stock_reserved
- stock_ordered
- stock_on_transfer

B) Calculate:
available_base_units =
    stock
    - stock_reserved
    + stock_ordered

For V1:
- Do NOT include stock_on_transfer in the math yet
- Show it in the UI only

C) Forecast short-term demand using historical weekday averages from invoice history

D) Calculate projected stock at receipt:
projected_units_at_receipt =
    available_base_units - pre_receipt_forecast_units

E) Calculate how much the next cycle needs to cover:
- Tuesday run (receipt Thursday) should cover:
  Friday + Monday + Tuesday
- Friday run (receipt Tuesday) should cover:
  Wednesday + Thursday

F) Add safety stock:
- default = 1 day of average forecast
- allow per-item override later

G) Order need:
raw_needed_units =
    max(0, cycle_forecast_units + safety_stock_units - projected_units_at_receipt)

H) Convert to cases:
suggested_cases = ceil(raw_needed_units / case_qty_units)
suggested_units = suggested_cases * case_qty_units

14. PLANNER LOGIC
==================================================

Create planner.py

Main function:
generate_replenishment_run(supplier_code, run_date, run_type, include_today_demand, current_user=None)

Flow:

Step 1:
Load supplier row from replenishment_suppliers.
If supplier not found or inactive, error.

Step 2:
Get receipt date, pre-receipt dates, and cover dates from calendar.py.

Step 3:
Call PS365 stock snapshot for the supplier using supplier_code and warehouse store 777.

Step 4:
From returned item codes, load:
- ps_items_dw master data
- replenishment_item_settings overrides
- weekday sales averages for all weekdays needed
- expiry summary

Step 5:
For each item:
- item_name = ps_items_dw.item_name else API item_name
- stock_now_units = API stock
- reserved_now_units = API reserved
- ordered_now_units = API ordered
- on_transfer_now_units = API on_transfer

available_base_units =
    stock_now_units - reserved_now_units + ordered_now_units

pre_receipt_forecast_units =
    sum of weekday forecast for pre-receipt dates

projected_units_at_receipt =
    available_base_units - pre_receipt_forecast_units

cover_forecast_units =
    sum of weekday forecast for cover dates

Resolve case_qty_units:
- first ps_items_dw.min_order_qty
- if invalid, fallback replenishment_item_settings.case_qty_units
- if still invalid => warning CASE_QTY_MISSING and suggested = 0

Resolve min_order_cases:
- from replenishment_item_settings.min_order_cases if present
- else default 1

Resolve safety_days:
- default 1.0
- override from replenishment_item_settings.safety_days_override if present

avg_daily_cover_forecast =
    cover_forecast_units / number_of_cover_dates

safety_stock_units =
    avg_daily_cover_forecast * safety_days

raw_needed_units =
    max(0, cover_forecast_units + safety_stock_units - projected_units_at_receipt)

If case_qty_units valid and raw_needed_units > 0:
    suggested_cases = ceil(raw_needed_units / case_qty_units)
    suggested_cases = max(suggested_cases, min_order_cases)
    suggested_units = suggested_cases * case_qty_units
Else:
    suggested_cases = 0
    suggested_units = 0

final_cases = suggested_cases
final_units = suggested_units

Step 6:
Build warnings:
- CASE_QTY_MISSING if no valid case qty
- NEGATIVE_PROJECTED_STOCK if projected_units_at_receipt < 0
- HAS_CURRENT_RESERVED if reserved_now_units > 0
- HAS_ORDERED_STOCK if ordered_now_units > 0
- EXPIRY_SOON if earliest_expiry_date within 30 days and qty > 0
- ZERO_ORDER if suggested_cases = 0

It is acceptable to store only one warning_code and one warning_text in V1.
If multiple conditions exist, prefer this priority:
1. CASE_QTY_MISSING
2. NEGATIVE_PROJECTED_STOCK
3. EXPIRY_SOON
4. HAS_ORDERED_STOCK
5. HAS_CURRENT_RESERVED
6. ZERO_ORDER

Step 7:
Create explanation_text like:
"Available base = stock 1126 - reserved 0 + ordered 0 = 1126. Pre-receipt forecast 180 => projected at receipt 946. Cover forecast 240 + safety 80 => raw need 0."

Store calc_json with:
- pre_receipt_dates
- cover_dates
- weekday averages used
- all numeric components

Step 8:
Save replenishment_runs header
Save replenishment_run_lines detail rows

Return run_id
