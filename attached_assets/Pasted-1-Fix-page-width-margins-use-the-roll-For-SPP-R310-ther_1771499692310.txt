But if you do this, you must also reduce/adjust COLS — otherwise padding and separators won’t match the real width.

3) IMPORTANT: compute COLS automatically from real width

Do not hardcode COLS=42 or 48 for PDF. Compute it from page width + font size using ReportLab’s font metrics.

Tell Replit to add this inside the PDF function (after defining PAGE_W/LEFT/RIGHT/FS):

from reportlab.pdfbase import pdfmetrics

avail_w = PAGE_W - LEFT - RIGHT
char_w = pdfmetrics.stringWidth("M", FONT, FS)   # monospace char width
COLS = max(30, int(avail_w / char_w))            # dynamic cols
sep = "-" * COLS


Then use COLS everywhere:

_pad_right(..., width=COLS)

wrap textwrap.wrap(..., width=COLS)

separator lines sep

This will automatically “extend the width” and keep alignment correct when you change FS.

6) Fix “Stop: 1.00” (formatting bug you should correct now)

Your printed receipt shows Stop: 1.00 which means you’re passing a float.

Force it to a 3-digit string:

try:
    stop_no = str(int(float(stop_no))).zfill(3)
