Refactor /admin/sms/ into a Messaging home / launcher / monitoring page.
Keep the route for now if needed, but change the page wording and behavior so it is no longer an “SMS-only standalone tool”.

Goal

This page should become the entry point for messaging-related work, while actual composition and sending continue to happen in the unified composer and bulk messaging flows.

This page should allow:
searching/selecting a customer to start a message
accessing templates
reviewing recent SMS delivery logs
checking provider balance
optionally jumping to bulk messaging
This page should NOT:
act like a full compose screen
send messages directly from this page
bypass the unified composer
