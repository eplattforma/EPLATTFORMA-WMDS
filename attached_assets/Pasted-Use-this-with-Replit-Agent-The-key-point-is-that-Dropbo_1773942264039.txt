Use this with Replit Agent. The key point is that Dropbox access tokens are short-lived, and background access should use OAuth code flow with a refresh token. Dropbox returns a refresh token when the authorization request includes token_access_type=offline, and new access tokens should then be obtained via /oauth2/token using grant_type=refresh_token. Dropbox also recommends scoped apps with the least permissions needed.

For Replit, keep the app key/secret in Secrets. Replit Secrets are encrypted and exposed to the app as environment variables. For recurring imports, Replit Scheduled Deployments are appropriate for periodic jobs and are explicitly meant for scheduled tasks rather than long-running web services.
