2) Create a service that reads from the existing abandoned carts data

You already have blueprints.abandoned_carts and it shows data, so your DB already has some table/view with the cart totals.

What to do

Locate the model/table your abandoned carts page uses. It will be in one of:

blueprints/abandoned_carts.py

a model like AbandonedCart, MagentoAbandonedCart, CartReport, etc.

a query reading from a table

Then implement a function:

services/crm_abandoned_cart.py
from datetime import datetime, timezone
from decimal import Decimal
from app import db
from models import PSCustomer, CrmAbandonedCartState

def upsert_abandoned_cart_state(customer_code_365: str, has_cart: bool, amount: Decimal | None, err: str | None = None):
    row = CrmAbandonedCartState.query.filter_by(customer_code_365=customer_code_365).first()
    if not row:
        row = CrmAbandonedCartState(customer_code_365=customer_code_365)
        db.session.add(row)

    cust = PSCustomer.query.filter_by(customer_code_365=customer_code_365).first()
    row.magento_customer_id = getattr(cust, "magento_customer_id", None)

    if err:
        row.sync_status = "FAIL"
        row.last_error = err[:500]
        row.last_synced_at = datetime.now(timezone.utc)
        db.session.commit()
        return

    row.has_abandoned_cart = bool(has_cart)
    row.abandoned_cart_amount = amount
    row.sync_status = "OK"
    row.last_error = None
    row.last_synced_at = datetime.now(timezone.utc)
    db.session.commit()

B) After refresh, query the existing dataset for ONE customer

Add a function:

from decimal import Decimal
from services.crm_abandoned_cart import upsert_abandoned_cart_state
from blueprints.abandoned_carts import refresh_abandoned_carts_data
from models import PSCustomer

def refresh_abandoned_cart_for_customer(customer_code_365: str):
    # 1) refresh global dataset (reuse existing logic)
    refresh_abandoned_carts_data()

    # 2) lookup customer mapping
    cust = PSCustomer.query.filter_by(customer_code_365=customer_code_365).first()
    if not cust or not getattr(cust, "magento_customer_id", None):
        upsert_abandoned_cart_state(customer_code_365, False, None, err="Missing magento_customer_id")
        return

    magento_customer_id = cust.magento_customer_id

    # 3) query YOUR existing abandoned carts table/model
    # Replace AbandonedCartModel + fields with your actual model/columns.
    row = AbandonedCartModel.query.filter_by(customer_id=magento_customer_id).first()

    if not row:
        upsert_abandoned_cart_state(customer_code_365, False, None)
        return

    # amount field name depends on your schema:
    amount = Decimal(str(row.cart_total or row.grand_total or row.amount or 0))
    upsert_abandoned_cart_state(customer_code_365, True, amount)

Important: You only store YES/NO + amount in CrmAbandonedCartState. No items, no history.

4) Add the manual refresh route (POST)

Wherever your customer view route is (CRM customer page or customer analytics page), add:

POST /crm/customer/<customer_code_365>/refresh-abandoned-cart

from flask import redirect, url_for, flash
from flask_login import login_required
from services.crm_abandoned_cart_refresh import refresh_abandoned_cart_for_customer

@crm_bp.post("/customer/<customer_code_365>/refresh-abandoned-cart")
@login_required
def refresh_customer_abandoned_cart(customer_code_365):
    try:
        refresh_abandoned_cart_for_customer(customer_code_365)
        flash("Abandoned cart refreshed.", "success")
    except Exception as e:
        flash(f"Abandoned cart refresh failed: {e}", "danger")
    return redirect(url_for("crm.customer_view", customer_code_365=customer_code_365))

6) Why this is the best approach (for your exact requirement)

Uses the abandoned cart pipeline you already trust (since your page already shows carts)

Customer page is always fast and does not depend on Magento latency/Cloudflare

Manual refresh gives you “fresh when needed”, but still controlled

Stores only what you asked: flag + amount (no history)
