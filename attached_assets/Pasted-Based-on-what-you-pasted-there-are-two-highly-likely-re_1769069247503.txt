D) Ensure the DB column exists (common root cause)

If ReceivingSession.comments is not a real DB column, session.comments = ... will never persist.

Run a quick DB check (Postgres):

SELECT column_name
FROM information_schema.columns
WHERE table_name = 'receiving_session'
  AND column_name = 'comments';


If it returns zero rows, add the column:

ALTER TABLE receiving_session ADD COLUMN comments TEXT;


(Adjust table name if yours is plural.)
