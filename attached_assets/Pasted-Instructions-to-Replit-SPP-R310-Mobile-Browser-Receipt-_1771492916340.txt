Instructions to Replit: SPP-R310 Mobile Browser Receipt (Seamless + Correct Content)
0) Core constraints (design decision)

Because drivers use Chrome on iOS (browser), we cannot do true Bluetooth direct print from the web app. The seamless method is:

✅ Generate 80mm thermal PDF server-side
✅ Use Web Share (preferred) to send the PDF to the BIXOLON printing app
✅ Fallback to open PDF in new tab
✅ Use signed short-lived token so the PDF works even if iOS loses session

A) Receipt Content Rules (must implement)
A1) Document type must match payment reality (no contradictions)

If payment NOT collected (payment_method == NOT_COLLECTED OR amount_collected <= 0):

E) Frontend (Mobile Web): One tap printing (Web Share)
E1) Preferred: Web Share with file

On Print button:

fetch(print_url) → blob

create File (receipt.pdf)

navigator.share({ files:[file] })

Fallback:

window.open(print_url, "_blank")

Replit should implement this helper:
async function shareOrOpenPDF(printUrl, filename) {
  try {
    const res = await fetch(printUrl, { cache: "no-store" });
    const blob = await res.blob();
    const file = new File([blob], filename, { type: "application/pdf" });

    if (navigator.canShare && navigator.canShare({ files: [file] })) {
      await navigator.share({ files: [file], title: filename });
      return;
    }
  } catch (e) {}
  window.open(printUrl, "_blank");
}

E2) Seamless UX toggle: Auto-print

Add driver setting:

Auto-print after completion (localStorage is fine)
If enabled:

After stop completion returns print_url, call shareOrOpenPDF() automatically.

F) One-time setup guidance inside the app (reduce support)

Create a “Printer Setup” page:

Print “Test Receipt” (open/share a sample PDF)

G) Edge cases (must handle)

Session lost / redirect → token solves it.

€ symbol wrong → add config to print EUR instead of €.

Long customer names → wrap to 48 columns; never wrap IDs.

Multiple invoices → list per line.

Reopened stop → show *** AMENDED DELIVERY *** at top.
