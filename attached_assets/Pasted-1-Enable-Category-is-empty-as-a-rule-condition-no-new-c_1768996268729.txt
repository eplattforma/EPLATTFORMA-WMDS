3) How you build the “No Category defaults” using rules

Because each Dynamic Rule sets one attribute, you create a bundle of rules that all share the same condition:

Condition (for all of them):

category_code_365 → is_empty

Then actions like:

zone = MAIN (confidence e.g. 70)

fragility = SEMI (confidence 65)

pressure_sensitivity = medium

temperature_sensitivity = normal

spill_risk = false

box_fit_rule = MIDDLE

Example condition JSON stored in DB:

{"all":[{"field":"category_code_365","op":"is_empty","value":""}]}


(Note: UI will submit empty value; your validator already allows blank value for is_empty.)

Recommended starting “No Category” bundle

To be conservative (reduce damages), I recommend:

fragility: SEMI

pressure: medium

spill_risk: false

temp: normal

box_fit: MIDDLE

zone: MAIN

Then add more specific higher-priority rules above it (same “category empty” + keywords/brand/attr1) for exceptions, e.g.:

name contains “SPRAY” or “BLEACH” → spill_risk=true, box_fit=BOTTOM

name contains “CHIPS” → pressure=high, box_fit=TOP, zone=SNACKS

Use priority to control precedence (higher wins).

4) You also mentioned filtering: show “No Category” in filters without adding a category

You can do this purely in the UI/backend, without creating a category record:

Template: templates/admin/oi/items.html

Add an option:

<option value="__EMPTY__" {% if filters.category == '__EMPTY__' %}selected{% endif %}>
  (No Category)
</option>

Backend: routes_oi.py inside oi_items()

Replace:

if category:
    query = query.filter(DwItem.category_code_365 == category)


With:

if category == '__EMPTY__':
    query = query.filter(or_(DwItem.category_code_365 == None, DwItem.category_code_365 == ''))
elif category:
    query = query.filter(DwItem.category_code_365 == category)


This solves the “blank does not appear in filtering” problem without adding any category.
