Replit change request — replenishment logic

The current replenishment logic appears to forecast demand by summing same-weekday averages over the next 7 cover dates. That is not aligned with the current business rule.

New business rule

Replenishment must ignore calendar date patterns and simply maintain approximately 7 days of stock forward.

Required changes

Remove weekday-based cover-date forecasting from replenishment

Do not forecast by looking at the weekdays of the next 7 dates.

Do not use last 4 same-weekday averages as the main replenishment driver.

Use recent weekly demand history instead

Build complete weekly buckets per SKU.

Fill missing weeks with zero.

Use last 26 weeks for classification.

Choose replenishment base forecast by demand class

smooth → MA8

intermittent / lumpy → Median6

new_sparse → simple recent non-zero average + review flag

no_demand → 0 + review if stock exists

Convert weekly forecast to daily forecast

daily_forecast = weekly_forecast / 7

Target stock formula

target_stock = daily_forecast × (cover_days + lead_time_days + review_cycle_days) + safety_stock

Recommended order

recommended_order = max(0, target_stock - net_available_qty)

Rounding

respect MOQ

respect case qty / order multiple

Safety stock

check current calculation for SNA-0118

if safety stock equals one day of forecast, do not label it as statistical safety stock

either:

rename it to buffer_stock_qty

or implement a real safety stock formula

remove run type from replenishment if only one replenishment policy now exists

keep separate admin actions for sync / rebuild if needed, but not as replenishment run types
