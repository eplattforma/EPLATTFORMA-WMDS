Change request: refocus Offer Intelligence from “margin risk” to “offer usage and offer sales dependency”

Business correction

For our business, these customer offers are intentionally low-margin in many cases because they are used to win, retain, or grow the customer.
So the CRM dashboard should NOT primarily highlight margin risk.

Instead, the dashboard and review-ordering screens should focus on:

1) how many active offers the customer has
2) what % of those offers are actually used
3) what % of the customer’s sales are on offered items

Margin can still exist in the drawer as secondary/commercial detail, but it should not be the main dashboard signal.

==================================================
1. NEW PRIMARY METRICS
==================================================

Implement these as the main offer metrics.

A) active_offer_skus
Definition:
- count of distinct active offered SKUs for the customer

B) offered_skus_bought_4w
Definition:
- count of distinct active offered SKUs that the customer bought in the last 28 days

C) offer_usage_pct
Definition:
- offered_skus_bought_4w / active_offer_skus * 100

Interpretation:
- how much of the offered assortment is actually being used

D) offer_sales_4w
Definition:
- total sales value in last 28 days from SKUs that are currently active offers for this customer

E) total_customer_sales_4w
Definition:
- total customer sales value in last 28 days from all sales, not only offers

Source:
- use the same sales base already used in dashboard/review-ordering value_4w logic where possible

F) offer_sales_share_pct
Definition:
- offer_sales_4w / total_customer_sales_4w * 100

Interpretation:
- what % of the customer’s sales are on offered items

4. SUMMARY REBUILD LOGIC CHANGES
==================================================

Update _rebuild_customer_offer_summary().

In addition to existing metrics, calculate:

A) total_customer_sales_4w
For each customer_code_365 in crm_customer_offer_current, pull total customer sales for the last 28 days from the same sales fact source used by CRM.
Preferred source:
- dw_sales_lines_mv
Fallback:
- whatever current CRM uses for value_4w if equivalent line-level sales source exists

Definition:
- sum(net_amount) for that customer over last 28 days

B) offer_usage_pct
Formula:
- if active_offer_skus > 0:
    offered_skus_bought_4w / active_offer_skus * 100
  else 0

C) offer_sales_share_pct
Formula:
- if total_customer_sales_4w > 0:
    offer_sales_4w / total_customer_sales_4w * 100
  else 0

Important:
Guard against divide-by-zero.
