# Add-on Brief: Cooler Route List + A4 Print Route Status Report

## Objective

Keep `/cooler/route-list` as the Warehouse Cooler Packing & Handover overview page, and improve the **Print Route Status** report so it becomes a clean A4 warehouse handover document.

The print report must work at every stage:

* before boxes are planned;
* during picking;
* when boxes are open;
* when all boxes are closed;
* after the cooler route is ready for dispatch.

---

# 1. Keep `/cooler/route-list`

Keep this link:

```text
/cooler/route-list
```

It is necessary as the **Warehouse Cooler Packing & Handover overview page**.

It must not be used as the picker screen.

The actual picker screen remains:

```text
/picker/batch/item/<batch_id>
```

The detail/handover page remains:

```text
/cooler/route/<route_id>/<delivery_date>
```

---

# 2. Rename `/cooler/route-list`

Rename the visible page title from:

```text
Cooler Routes Overview
```

to:

```text
Cooler Packing & Handover
```

Add subtitle:

```text
Warehouse overview of cooler routes, box planning, picking progress and dispatch handover.
```

---

# 3. Add Warehouse menu item

In the Warehouse menu, add a visible menu item:

```text
Warehouse → Cooler Packing
```

Link:

```python
url_for('cooler.route_list')
```

Make sure the Warehouse menu active state includes:

```text
cooler.route_list
cooler.route_picking
cooler.route_manifest
cooler.box_label
cooler.box_manifest
```

Optional alias:

```text
/warehouse/cooler
```

This can redirect to:

```text
/cooler/route-list
```

---

# 4. `/cooler/route-list` data requirements

The page must show **all route/date cooler work**, not only pending work.

It must include:

```text
routes needing planning
routes being picked
routes with open boxes
routes ready for dispatch
completed closed boxes for today
```

Use both data sources:

```text
batch_pick_queue
cooler_boxes
cooler_box_items
```

Do not build the overview only from `batch_pick_queue`, because completed closed cooler boxes may disappear from the overview.

Group by:

```text
route_id
delivery_date
```

---

# 5. `/cooler/route-list` row columns

Each row should show:

```text
Route
Delivery Date
Driver
Cooler Items
Picked / Total
Boxes Closed / Total
Open Boxes
Label Status
Route Status
Actions
```

Suggested status rules:

```text
Needs Planning
- cooler items exist
- no cooler boxes exist

Picking In Progress
- pending cooler queue items exist

Needs Boxing
- picked cooler items exist but are not assigned to boxes

Boxes Open
- at least one non-cancelled cooler box is open

Ready for Dispatch
- all cooler queue items picked
- all non-cancelled boxes closed
- no picked/unboxed items
- no open boxes

Exception
- any cooler queue item has exception status
```

Actions per row:

```text
Open Handover
Print Route Manifest
Print Route Status
Print All Labels
```

---

# 6. Improve Print Route Status report

## Preferred approach

Create a dedicated print route instead of relying only on screen CSS.

Add endpoint:

```text
/cooler/route/<route_id>/<delivery_date>/status-print
```

Suggested Flask route name:

```python
cooler.route_status_print
```

The existing “Print Route Status” button should open this dedicated print view.

This report should be printable directly from browser and optimised for A4.

---

# 7. Print Route Status report purpose

The report should answer:

```text
Which cooler items belong to this route?
Which stop/customer/invoice do they belong to?
Have they been picked?
Which box are they in?
Is the box open or closed?
Are the cooler boxes ready for dispatch?
```

It should be useful for:

```text
warehouse manager
picker supervisor
driver handover
dispatch checking
reprinting after completion
```

---

# 8. A4 format

Use A4 landscape as the default because the item status table is wide.

CSS:

```css
@page {
  size: A4 landscape;
  margin: 8mm;
}
```

General print style:

```css
body {
  font-family: Arial, Helvetica, sans-serif;
  font-size: 9pt;
  color: #000;
  background: #fff;
}

.no-print {
  display: none !important;
}

.print-only {
  display: block !important;
}

table {
  width: 100%;
  border-collapse: collapse;
}

thead {
  display: table-header-group;
}

tr {
  page-break-inside: avoid;
  break-inside: avoid;
}

th, td {
  border: 1px solid #444;
  padding: 3px 4px;
  vertical-align: top;
}

th {
  font-weight: bold;
  background: #eee;
}
```

Do not rely only on colours. Use text labels like:

```text
Picked
Not Picked
Open
Closed
Unboxed
Exception
```

# 17. Page breaks

Avoid breaking inside:

```text
box summary rows
stop group headers
item rows
```

CSS:

```css
.stop-group-header {
  page-break-after: avoid;
  break-after: avoid;
  font-weight: bold;
  background: #ddd;
}

.stop-group {
  page-break-inside: avoid;
  break-inside: avoid;
}
```

However, if a stop has many items and cannot fit on one page, allow it to continue, but repeat table headers.

Use:

```css
thead {
  display: table-header-group;
}
```

# 24. Acceptance criteria

This implementation is complete when:

1. `/cooler/route-list` remains available.
2. Page title is renamed to `Cooler Packing & Handover`.
3. Warehouse menu has visible `Cooler Packing` entry.
4. `/cooler/route-list` shows active and completed cooler routes.
5. `/cooler/route-list` uses both `batch_pick_queue` and `cooler_boxes`.
6. Completed closed cooler boxes still appear after picking is done.
7. Each route row has `Open Handover`, `Print Route Manifest`, `Print Route Status`, and `Print All Labels`.
8. `Print Route Status` opens a dedicated A4-optimised print view.
9. Print report works before boxes are planned.
10. Print report works during picking.
11. Print report works after all boxes are closed.
12. Print report uses A4 landscape.
13. Print report shows route header, driver, delivery date, printed timestamp and printed by.
14. Print report shows compact KPI summary.
15. Print report shows box summary / dispatch handover.
16. Print report shows loading guidance when boxes are closed.
17. Print report shows Route Item Status grouped by stop.
18. Print report does not show Customer Code or Queue ID.
19. Long item descriptions wrap cleanly.
20. Table headers repeat on each printed page.
21. Buttons/navigation are hidden in print.
22. The printout is readable on A4 without cutting off columns.
