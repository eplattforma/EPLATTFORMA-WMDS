Bug Fix + Enhancement: Cooler picking screen — picker should use batch interface, not cooler screen Pick buttons
Issue 1 — Pick buttons should be hidden when a batch is in progress
When the "Picking in progress" banner is showing (i.e. COOLER-ROUTE-<id> batch exists and status is not Completed), the Action column Pick buttons should be removed from the Sequenced table.
Instead of a Pick button, each row should show a greyed-out indicator:
Status column: [⏳ Awaiting batch pick]
Action column: — (no button)
This forces the picker to use the correct batch interface. The cooler screen Pick buttons are only useful as an emergency fallback when there is no batch — they should not be the primary interface.
In templates/cooler/route_picking.html, add a condition:
html{% if not batch_in_progress %}
  <td><button class="btn btn-sm btn-primary pick-btn">Pick</button></td>
{% else %}
  <td><span class="text-muted small">via batch</span></td>
{% endif %}
Pass batch_in_progress as a template variable from the route_picking view in blueprints/cooler_picking.py. Set it to True when a non-completed cooler session exists for this route.
