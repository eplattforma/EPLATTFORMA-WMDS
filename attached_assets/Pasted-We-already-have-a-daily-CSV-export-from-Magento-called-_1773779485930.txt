We already have a daily CSV export from Magento called:

customer_price_master.csv

This file contains customer-specific price offers from Magento/BSS custom pricing.
We want to import it into Replit and use it later on the CRM customer screen.

Goal
We want CRM to understand what special prices/offers a customer gets, link them to our existing customer and item master data, and present them clearly on the customer profile.

Source file columns
The CSV contains these columns:

- snapshot_at
- customer_id
- customer_email
- sku
- product_name
- rule_code
- rule_name
- rule_description
- origin_price
- customer_final_price

Important business meaning
- customer_id = Magento customer id
- sku = Magento product sku
- this file represents special/customer pricing offers
- we want to link:
  - customer_id -> our PSCustomer record using Magento customer id
  - sku -> our item master using item code / item_code_365
- then show the grouped information on the CRM customer page

Please implement this in Replit using our normal Flask + SQLAlchemy structure.

3. Import logic
Build an importer that:
- reads customer_price_master.csv
- validates required columns
- loads rows into crm_customer_price_offer_import
- then transforms/upserts into crm_customer_price_offer

Normalization rules:
- trim all strings
- uppercase sku before matching
- convert prices to decimal
- discount_amount = origin_price - customer_final_price
- discount_percent = ((origin_price - customer_final_price) / origin_price) * 100 when origin_price > 0

6. Refresh strategy
This file is a daily full snapshot of current offers.

So the normalized final table should behave like a current-state table.

Recommended approach:
- import raw rows into crm_customer_price_offer_import with import_batch_id
- then upsert into crm_customer_price_offer by (magento_customer_id, sku)
- after successful import, remove rows from crm_customer_price_offer that were not present in the latest batch
  OR replace the whole final table from the latest batch if simpler/safer

Choose the safest implementation for our stack.
