Please update the picking/batch screen table layout to make it more compact and easier to read on narrow screens.

This is a UI/display change only. Do not change the underlying database values or business logic unless needed to calculate display status.

====================================================
1. COMBINE TIME COLUMNS
====================================================

Current columns:
- Est. Time
- Actual
- Remaining

Replace them with one column called:

Time

The new Time column should display two lines:

Line 1:
- Actual time if available.
- Use the current blue badge style used for actual time.
- If there is no actual time, show "—".

Line 2:
- Estimated time only.
- Format: Est Xm
- Do not show remaining time.
- Do not show comments such as "over by", "left", "not started", or "slightly over".
- Do not use a background badge for the estimate.
- Estimate should be plain coloured text on the dark row background.

Example displays:

Actual exists and is over estimate:
50m
Est 6m   ← red text

Actual exists and is on time:
6m
Est 8m   ← green text

No actual yet:
—
Est 5m   ← grey text

Estimate font colour rules:
- Grey: no actual time yet / not started.
- Green: actual time is less than or equal to estimated time.
- Orange: actual time is slightly over estimate.
- Red: actual time is significantly over estimate.

Suggested thresholds:
- Green: actual <= estimate
- Orange: actual > estimate and actual <= estimate * 1.25
- Red: actual > estimate * 1.25

====================================================
2. ROUND DISPLAY VALUES ON THIS SCREEN
====================================================

Round these values to 0 decimal points for display only:

- Weight
- Estimated minutes
- Actual minutes

Do not show decimals on this screen.

Examples:
- Weight 121.0 should display as 121
- Est. Time 6.0 should display as Est 6m
- Actual 50.26m should display as 50m
- Actual 5.5m should display as 6m
- Est. Time 15.0 should display as Est 15m

Important:
Only round for display. Do not update or round the stored database values.

====================================================
3. COMBINE LINES AND QTY COLUMNS
====================================================

Current columns:
- Lines
- Qty

Problem:
The current Qty column is actually showing picked lines / total lines, for example:
- 7/7
- 0/8
- 24/24

This is not really quantity, so the label Qty is misleading.

Change:
- Remove/hide the separate Lines column that only shows the total number.
- Rename the current Qty progress column to Lines.
- Display it as picked lines / total lines.

Examples:

Current:
Lines: 7
Qty: 7/7

New:
Lines: 7/7

Current:
Lines: 8
Qty: 0/8

New:
Lines: 0/8

Current:
Lines: 24
Qty: 24/24

New:
Lines: 24/24

Optional visual improvement:
- If picked lines = total lines, show the value in green or completed style.
- If picked lines = 0, show neutral/grey.
- If picked lines is between 0 and total lines, show blue/orange to indicate in progress.

====================================================
4. COMBINE CUSTOMER AND ROUTING / NOTES
====================================================

Create one compact column called:

Customer

Inside the Customer column, display two lines:

Line 1:
- Customer name.
- Larger, bold, readable font.
- Truncate with ellipsis if too long.

Line 2:
- Routing / route group / delivery note information.
- Smaller font, but still readable.
- Muted light grey colour.
- Truncate with ellipsis if too long.
- If there is no routing/notes value, hide the second line instead of leaving empty space.

Example:

Customer
EKO CYPRUS LIMITED
LEFKOSIA WED1 MOUNTAIN

Another example:

Customer
PETROLINA SERVICE STATION
LEFKOSIA WED1 MOUNTAIN · call before delivery

Important:

====================================================
5. FINAL TARGET COLUMN STRUCTURE
====================================================

The table should move toward this compact structure:

Status | Customer | Lines | Weight | Time | Action / Assigned

Where:

Customer:
- Line 1: Customer name
- Line 2: Routing / notes

Lines:
- picked lines / total lines
- example: 7/7, 0/8, 12/12

Weight:
- rounded with no decimals
- example: 121, 685, 48

Time:
- Line 1: actual minutes, rounded, blue badge or "—"
- Line 2: Est Xm, rounded, coloured font only
