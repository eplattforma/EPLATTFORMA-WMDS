If Step 7 Still Fails
If the Assign button still does nothing after these fixes, the issue has moved from qty_picked=0 to something else. The most likely remaining cause is the str(invoice_delivery_date) != str(box["delivery_date"]) comparison in box_assign_item().
Date formatting can differ between how Python stores it (datetime.date(2026, 5, 14) → str() gives '2026-05-14') and how it comes from the database ('2026-05-14 00:00:00' or '14/05/2026' depending on the query). A one-character format mismatch silently rejects every assignment.
If that's the case, add explicit formatting to both sides:
pythonfrom datetime import datetime

def _normalise_date(d):
    if d is None:
        return None
    if isinstance(d, str):
        # Try parsing common formats
        for fmt in ('%Y-%m-%d', '%Y-%m-%d %H:%M:%S', '%d/%m/%Y'):
            try:
                return datetime.strptime(d.split('T')[0], fmt).date()
            except ValueError:
                continue
    if hasattr(d, 'date'):
        return d.date()
    return str(d)
