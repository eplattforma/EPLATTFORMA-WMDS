1) Fix a real bug: COLS is calculated for 10pt, but titles print at 12pt

Right now:

cols = get_dynamic_cols() uses FS_NORMAL=10

but add_bc() prints with FS_TITLE=12

yet you truncate title strings to s[:cols] (10pt capacity)

This can still overflow/clamp at the sides on some printers/apps because 12pt characters are wider.

✅ Tell Replit to compute two column widths:

cols_normal for FS_NORMAL

cols_title for FS_TITLE

Patch
def get_dynamic_cols(font_name, font_size):
    avail_w = PAGE_W - LEFT - RIGHT
    char_w = pdfmetrics.stringWidth("M", font_name, font_size)
    return max(30, int(avail_w / char_w))

def build_delivery_receipt_pdf(data: dict) -> bytes:
    cols = get_dynamic_cols(FONT, FS_NORMAL)
    cols_title = get_dynamic_cols(FONT_B, FS_TITLE)
    sep = "-" * cols

    def add(s=""):
        lines.append(("L", s[:cols]))
    def add_b(s):
        lines.append(("B", s[:cols]))
    def add_c(s):
        lines.append(("C", s[:cols]))
    def add_bc(s):
        # IMPORTANT: truncate based on title font size
        lines.append(("BC", s[:cols_title]))


This alone usually makes the receipt look “wider” because the title lines stop getting squeezed/clipped.
