The core idea is:


Extend the existing classification job to support filtered reclassification (single SKU, category, or all).


Trigger that job automatically after admin changes.


Because classification.engine.reclassify_items() already calls classify_single_item() and upsert_packing_profile(), the refresh will update both tables correctly.

B. Create a helper to start the thread consistently
Add a small helper in routes_oi.py (near oi_reclassify()):
def _start_reclassify_thread(username, summer_mode, item_codes=None, category_code=None):
    running_job = _get_running_classification()
    if running_job:
        return False  # job already running

    thread = threading.Thread(
        target=_run_classification_background,
        args=(username, summer_mode, item_codes, category_code)
    )
    thread.daemon = True
    thread.start()
    return True
