Use item_cost from Powersoft365 instead of price_excl_1. In the Powersoft docs, the /list_items field list explicitly includes item_cost, item_average_cost, and item_weighetd_average_cost, and it describes price_excl_1 separately as selling price 1 excluding VAT. The /item docs also define item_cost as the purchased cost of the item. The /list_items docs further say request_password_365 is required if the user needs item cost.

Changes to make

Add request_password_365 to the list_items payload
In both places where we call list_items:

full_dw_update()

incremental_dw_update()

also test_fetch_single_item() if you want test coverage

Add this inside filter_define:

"request_password_365": os.environ.get("PS365_REQUEST_PASSWORD_365", ""),

This should be the special Powersoft request password for cost access, not the normal token. Store it in Replit Secrets as PS365_REQUEST_PASSWORD_365. The docs say this parameter is optional in general, but necessary when item cost is needed.

Add item_cost to display_fields
In every display_fields string currently using price_excl_1, add item_cost. I suggest also adding the other two cost fields for future analysis:

"display_fields": "item_code_365,item_name,active,category_code_365,brand_code_365,season_code_365,attribute_1_code_365,attribute_2_code_365,attribute_3_code_365,attribute_4_code_365,attribute_5_code_365,attribute_6_code_365,item_length,item_width,item_height,item_weight,number_of_pieces,number_field_1_value,number_field_2_value,number_field_5_value,text_field_2_value,list_item_barcodes,vat_code_365,vat_percent,item_cost,item_average_cost,item_weighetd_average_cost,price_excl_1,supplier_code_365,supplier_name"

The docs confirm all four fields are valid display_fields values.

Change DW mapping so cost_price comes from item_cost
In build_item_core() replace:

"cost_price": to_float(ps_item.get("price_excl_1")) or None,

with:

"cost_price": to_float(ps_item.get("item_cost")) or None,

This is the key fix. item_cost is the purchased cost; price_excl_1 is selling price excluding VAT.

Optional but recommended: store average-cost fields too
If the DwItem model does not already have fields for this, add nullable columns such as:

average_cost_price

weighted_average_cost_price

Then map them in build_item_core():

"average_cost_price": to_float(ps_item.get("item_average_cost")),
"weighted_average_cost_price": to_float(ps_item.get("item_weighetd_average_cost")),

Those fields are available in the item list field reference.

Do not remove price_excl_1 from the API response yet
Keep fetching it for now if other parts of the app may still use it as selling price. But do not map it into cost_price anymore. The docs distinguish them clearly.
