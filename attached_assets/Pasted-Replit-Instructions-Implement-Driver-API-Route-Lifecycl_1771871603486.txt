1.2 Fix services_route_lifecycle.compute_stop_status() (status vocab bug)

File: services_route_lifecycle.py

Replace the whole compute_stop_status() body with normalize-based logic:

from delivery_status import normalize_status

def compute_stop_status(route_stop_id: int):
    invoices = get_stop_active_invoices(route_stop_id)
    if not invoices:
        return "IN_PROGRESS"

    delivered = 0
    terminal_not_delivered = 0

    for i in invoices:
        s = normalize_status(i.status)
        if s == "delivered":
            delivered += 1
        elif s in ("delivery_failed", "returned_to_warehouse"):
            terminal_not_delivered += 1

    total = len(invoices)

    if delivered == total:
        return "DELIVERED"
    if terminal_not_delivered == total:
        return "FAILED"  # you can rename to "NOT_DELIVERED" later, keep UI stable for now
    if delivered + terminal_not_delivered > 0:
        return "PARTIAL"
    return "IN_PROGRESS"

This removes the old 'FAILED'/'RETURNED' mismatch.
