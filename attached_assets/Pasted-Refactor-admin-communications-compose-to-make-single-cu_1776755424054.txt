Refactor /admin/communications/compose to make single-customer SMS sending the primary workflow, while keeping Push / WhatsApp / Phone as secondary options. Do not change the backend send logic yet unless needed for the UI toggle behavior. This is mainly a UI/UX restructuring of the existing page.

Goals
Make SMS the default and dominant action.
Hide Push-specific fields unless Push is the selected channel.
Reduce clutter from customer info and history.
Keep all existing capabilities, but demote non-SMS actions.
1. Add a channel selector at the top of the compose area

Inside the main compose card, above template selection, add a compact channel selector such as:

SMS
Push
WhatsApp
Phone

Default selected channel:

SMS

Implementation guidance:

Use a simple segmented control / pill buttons / radio group.
Bind selected value in the page state, for example:
sms
push
whatsapp
phone

Behavior:

On initial page load, selected channel should default to sms.
If there is an explicit query param or server-side context already indicating push mode, allow preselecting push.
Otherwise always default to sms.
2. Show/hide sections based on selected channel
A. SMS mode

When selected channel = sms:

Show:
Template dropdown
Message textarea
SMS character count / parts
Main primary button = Send SMS
Hide:
Push title
Push click URL
Push destination / target type
Secondary actions may still exist, but do not visually compete with the primary send button.
B. Push mode

When selected channel = push:

Show:
Template dropdown if it is already supported for push-compatible text
Message body if used for push body
Push title
Push click URL
Push destination / target type
Primary button = Send Push
Hide SMS-specific wording such as “SMS parts” where not relevant.
C. WhatsApp mode

When selected channel = whatsapp:

Show:
Template dropdown
Message textarea
Primary button = Open WhatsApp or existing equivalent action
Hide push fields.
D. Phone mode

When selected channel = phone:

Show:
Customer phone number
Existing phone/send-via-phone action
Hide push fields.
Message area can remain visible if useful, but this mode should not look like an SMS send mode.
3. Make the primary action singular and obvious

Currently there are multiple same-weight buttons. Change that.

Required behavior

Only one primary action button should be visually dominant at a time, based on selected channel:

SMS → Send SMS
Push → Send Push
WhatsApp → Open WhatsApp
Phone → Send via Phone or existing equivalent wording
Secondary actions

Move other channels into a secondary area such as:

a small inline secondary action row
or a collapsible “Other actions”

Do not show all channel buttons as equal large buttons at the same time.

Important

If “MicroSMS” is the actual backend provider, keep it internally, but the UI label should be user-friendly:

Prefer Send SMS
Do not use Send MicroSMS as the main label unless there is a real business reason
