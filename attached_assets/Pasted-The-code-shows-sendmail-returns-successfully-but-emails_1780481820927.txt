The code shows sendmail returns successfully but emails don't arrive anywhere — not even the Gmail CC. This points to the SMTP relay accepting the message but failing silently on delivery, not a code logic issue.
Two things I notice in the code that could cause this:
Issue 1 — Missing critical email headers
The message is built without Date and Message-ID headers. Many corporate mail servers (and some relays) silently discard messages missing these — they're required by RFC 2822.
Issue 2 — sendmail return value is ignored
If some recipients fail (e.g. pepsico.com rejects), sendmail returns a dict of failed addresses but the code ignores it and logs "success". You can't tell what actually happened.
