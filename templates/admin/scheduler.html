How scheduling works on this deployment. Jobs only fire while a worker is awake. Because the production app runs on autoscale (workers spin down when there's no traffic), all daily jobs are intentionally scheduled in the late afternoon when warehouse staff are using the system. Sub-hourly jobs (pending orders, payment retries, FTP login) fire naturally throughout the business day.
Editing schedules: Changes you make here are written to the shared schedule database immediately. The running scheduler picks them up on its next wake cycle — usually within a few minutes — so you don't need to redeploy. Pause, Resume and Run Now all take effect right away.
