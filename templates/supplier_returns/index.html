How This Screen Works

Stock comes from PS365 store 100 (RETURNS). Click Refresh to sync. Page loads use the local database — no API call needed on navigation.

Cases vs Pieces: Cases = raw PS365 quantity. Pieces = cases × pack size (e.g. 0.07 cases × 15 = 1 piece).

New Collection: Select items for a supplier and click New Collection. This sends a Purchase Order to PS365 with status RETURN. The items show as "on PO" and are excluded from future collections.

Open Collections: Shown inside each supplier card. Outstanding = ordered − already received by PS365. Only POs with status RETURN assigned to store 100 with a RET- cart code are shown.

Mark Collected: Click when the supplier has physically taken the items. This records the handover locally. You must also process the Purchase Return in PS365 to deduct the stock — this system cannot do that automatically.

Count column: Enter the physical piece count to compare against the system. Diff shows instantly. Nothing is saved.

Search: Type any part of the supplier name or code to filter. Press Esc or click ✕ to clear. Collapse All folds all cards to headers for a quick overview.
