How This Screen Works
What you see

Items currently in Store 100 (RETURNS) that are awaiting collection by their supplier. Stock quantities are pulled from PS365 and cached in the database — click Refresh to sync from PS365.

Available vs On PO

If an item already has an open Return PO, its quantity is shown in two parts:

  • On PO — cases already committed on an open Return Purchase Order the supplier has not yet collected.
  • Available — remaining cases you can include in a new PO.
Excluded orders — what gets ignored

A Return PO is excluded (does not block stock) when any of the following apply:

  • Its status in PS365 is not RETURN — only POs with exactly status RETURN are treated as open. Statuses such as REC, CANCELLED, COMPLETED, or any other value are ignored.
  • Its store in PS365 is not store 100 — if PS365 assigned the PO to store 777 (or any store other than 100), it is excluded even if the status is RETURN.
  • Its cart code does not contain RET- — only POs created by this system (cart codes starting with RET- or WMDS-RET-) are considered.
Refresh

Refresh fetches live stock from PS365 and checks open Return POs. Page loads between refreshes use the database cache — instant, no PS365 call. Stock data survives a server restart.
