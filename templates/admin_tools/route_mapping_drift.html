About Route Mapping

The canonical source for invoice-to-route mapping is the route_stop_invoice table (where is_active = true).

For backward compatibility, invoices also have route_id and stop_id cache columns. These should always match the canonical mapping.

Drift occurs when these cache columns don't match the canonical mapping, which can cause inconsistent behavior in the UI.
